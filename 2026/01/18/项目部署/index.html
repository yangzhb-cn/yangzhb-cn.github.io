原生部署

前端部署

黑马点评作为参考

  • 安装目标软件或者直接本地上传
  • 现在的前端项目可以打包成dist然后上传
  • 需要web服务器:nginx、apache、tomcat
    • 注意nginx权限
    • nginx -t 检查配置
      • 将前端项目移动到 /opt/homebrew/Cellar/nginx/1.29.4/html/hmdp
      • 后面配置nginx的配置文件中的root时可以使用相对路径，当然不进行移动的话使用绝对路径
      • nginx的配置文件 /opt/homebrew/etc/nginx/nginx.conf
    • nginx -s reload 或重启 Nginx 使配置生效（如果只替换静态文件，一般不必重启）

后端部署

  • 安装jdk、maven
    • (不用maven也可以，本地打好依赖包后的jar包直接上传)
  • 安装解压后，找到bin目录下的mvn，为了方便可以修改环境变量
    • 先复制这个bin目录的绝对路径 /root/services/apache-maven-3.8.5/bin

    • PATH=...

      • PATH 是命令搜索路径列表（用冒号 : 分隔）。你敲 nginx、mvn 这种命令时，Shell 会按 PATH 里的目录顺序去找同名可执行文件。

    • $PATH

      • 代表当前已有的 PATH 值。把它放在最前面表示“保留原来的路径，再追加新的”。

    • :/usr/local/nginx/sbin

      • 追加 Nginx 的可执行文件目录（通常里面有 nginx）。这样你在任意目录都可以直接运行 nginx，不用写全路径 /usr/local/nginx/sbin/nginx。

    • :/root/services/apache-maven-3.8.5/bin

      • 追加 Maven 的 bin 目录（通常里面有 mvn）。这样可以直接运行 mvn。

    • export

      • 把设置后的 PATH 导出为“环境变量”，让当前 Shell 启动的子进程（例如你运行的 mvn、nginx）也能继承这个 PATH

    • 如果使用一些包管理器如yum、apt等则不需要配置环境变量

  • 将后端项目上传，或者直接在本地git然后再clone
  • 进入后端项目的根目录下，打jar包，或者直接在本地打好jar包后上传到git/本地上传。然后执行

宝塔部署

使用云服务器部署时，记得开放端口

  • 直接安装宝塔，然后在界面进行操作
    • 安装对应的java(安装Tomcat自带)、nginx
  • 有了宝塔，不用再自己手动改nginx操作
    1. 先进入网站，进入PHP，增加站点，只需要在域名处改动，这里改成ip
    2. 进入到这个站点目录，将默认文件删掉
    3. 将前端dist里面的文件全部上传到当前目录
    4. 然后宝塔自动帮我们搞定，直接访问这个域名也就是ip地址就可以了
  • 当然也可以直接在当前站点的设置按钮，查看或者修改nginx的配置文件
  • 对于后端项目
    1. 进入文件，从刚刚加入的前端项目目录下的同级目录增加一个新文件夹（后端项目名）
    2. 进入这个新文件夹，将在本地打包好的jar包放进去，记住这个绝对路径
    3. 进入网站，进入java项目，添加java项目，有个项目jar路径，按第3点填，项目名称对应，项目端口8080（有可能被宝塔安装的tomcat占用了，将其关掉即可，装tomcat只是为了装tomcat）
    4. 修改项目执行命令，还是那一个 java -jar <绝对路径>.jar 选项信息，然后就可以了
    5. 如果启动不了，可能是宝塔没有开放8080端口，在安全那里开放即可

Docker部署

  • docker是容器，可以将项目的环境(比如java、nginx)和项目的代码一起打包成镜像，镜像可以让其他人下载。
  • docker安装或者宝塔安装
  • 将项目制作成镜像

dockerfile

  • 首先需要一个文件dockerfile，用于指定构建docker镜像的方法。添加到项目的根目录
  • dockerfile不用完全自己写，直接网上如github查找相关的同类项目然后在他的基础上改即可，这里是后端项目
  • 对于前端的dockerfile，需要增加nginx的配置
    • 在前端项目根目录新建一个docker目录（存放docker需要的文件），增加文件nginx.conf
    • 前端的dockerfile

制作镜像

  • 进入到项目目录中，此时ls后看到有dockerfile文件

docker 构建优化

  • 最好是把构建好的项目上传，而不是在dockerfile里面写
  • 减少尺寸、减少构建时间（多构建构建，如将maven打包和启动分成两个构建，可以丢弃之前阶段不需要的命令）

绑定域名与跨域

绑定域名

  • 首先需要有一个域名解析服务，添加一个解析记录，腾讯云/阿里云

    • 将域名指向一个ip地址

  • 前端项目访问:

    • 用户输入网址
    • 域名解析服务器(把网址解析为ip地址/交给其他的域名解析服务)
    • 服务器
    • 防火墙
    • nginx接收请求,找到对应的文件,返回文件给前端
      • 宝塔面板配置nginx，从域名管理那里需要添加域名（否则Nginx不认识这个域名）修改server_name
    • 前端加载文件到浏览器中(js、CSS)
    • 渲染页面

  • 后端项目访问:

    • 用户输入网址
    • 域名解析服务器
    • 服务器
    • ngin×接收请求
      • 对于后端项目，直接写域名访问后端项目，还需要增加端口
      • 每次加端口号才能访问太丑了，需要nginx帮忙转发
    • 后端项目(比如8080端口)
    • nginx反向代理:替服务器接收请求,转发请求
      • 对于宝塔部署，重新创建一个项目，修改nginx配置，添加一个反向代理
      • 目标URL ：http://IP :8080
      • 发送域名：默认
      • 这个时候访问后端项目域名的根目录会自动转发到目标URL上

跨域

  • 此时如果访问前端页面，在前端页面访问后端接口，会产生跨域问题
  • 原因是浏览器为了用户的安全,仅允许向同域名、同端口的服务器发送请求
    • 为了检测跨域，浏览器会在发送请求之前，发送一个预检测请求（提前探路），请求方法OPTIONS

解决跨域问题

  • 将域名、端口改成相同的
    • 比如nginx绑定前端的域名和端口，当访问/api时，将请求转发到后端域名+端口即可
  • 让服务器告诉浏览器：运行允许跨域（返回cross-origin-allow响应头）
    • 可以让nginx告诉浏览器，也可以让后端服务器告诉浏览器
    • 网管支持（nginx）
  • 修改后端服务
    • 返回新的CorsFilter Bean
    • 重写WebMvcConfigurer(添加web全局请求拦截器)，项目新建一个config目录，写进去就可以了
    • 在controllor层，类名上面补充注解，也有参数可以选择
    • 手工设置响应头（HttpServletResponse ）
  • 以上是设置cors跨域后端解决的四种方式，本质都是类似最后一种设置响应头，不过都是各自包实现了不同的封装逻辑。

常用终端命令(自查)

发表于2026-01-15|更新于2026-01-15

Linux命令

符号

> >> | & FD

  • > 把标准输出（stdout）重定向到文件，覆盖写入（文件不存在则创建）
  • >>把标准输出追加到文件末尾，不覆盖
  • |把左边命令的输出作为右边命令的输入，用于把多个命令串起来处理数据流
  • &把命令放到后台运行，终端不会被该命令阻塞，你可以继续输入其他命令
  • &放在重定向里（如 2>&1）表示“后面跟的是文件描述符FD”，用于把一个 FD 重定向到另一个 FD
  • FD操作系统为进程打开的每个输入/输出通道分配的一个整数编号，用来读写文件、终端、管道、网络等资源

文件

mkdir touch rm cp mv

  • mkdir创建文件夹/目录
  • touch 创建一个空的新文件
  • rm 删除一个文件/文件夹
  • cp 复制文件或目录，用于备份或批量复制
  • mv 移动文件/目录，或重命名（移动当前目录）

cat more less head tail

  • cat 显示整个文件内容
  • more 逐页显示文件内容，空格向下翻页
  • less与 more类似,但提供更多功能,如向前/向后翻页、搜索等
  • head 显示文件的开头部分,默认显示头部10行
  • tail查看实时滚动日志

grep wc sed echo

  • grep 在文件或输入中按字符串模式搜索匹配的行，常用于日志/文本检索
  • wc 统计文本的行数/单词数/字节数（或字符数）
  • sed 替换文件中的字符串
  • echo 把字符串或变量内容输出到标准输出，常用于打印信息、生成简单文本或配合重定向写文件

ls du stat pwd find

  • ls 显示文件详细信息
  • du显示目录或文件的大小,以人类可读的方式(例如KB、MB)显示
  • stat显示文件的详细信息,包括文件大小和其他属性
  • pwd显示当前工作目录的绝对路径
  • find在目录树中按条件查找文件/目录（支持按名称、类型、时间、大小等过滤）

sudo chmod

  • sudo 以其他用户（默认 root）权限执行命令，用于临时提升权限执行需要管理员权限的操作
  • chmod修改文件/目录权限，常用数字模式或符号模式设置读/写/执行权限

apt tar zip gzip

  • apt 用于安装、更新、升级和卸载软件包
  • tar 用于打包与解包（常配合 gzip 压缩）
  • zip/unzip 用于 zip 格式的压缩与解压
  • gzip/gunzip 用于单文件 gzip 压缩与解压

CPU、内存、磁盘

ps top free df uptime kill

  • ps 一次性查CPU等占用
  • top 实时排查CPU等占用
  • free查询操作系统内存使用
  • df查看磁盘空间
  • uptime查看当前系统负载情况
    • Load Average的三个数字,依次则是过去1分钟、5分钟、15分钟的平均负载。可以通过观察这三个数字的大小,可以简单判断系统的负载是下降的趋势还是上升的趋势。
      • 负载值一般不超过cpu核数的1-1.5倍,如果超过1.5倍,那就要重视,此时会严重影响系统。
      • 平均负载是指单位时间内,处于可运行状态和不可中断状态的为进程数。所以,它不仅包括了正在使用CPU的进程,还包括等待CPU和等待 I/O的进程。
      • 而CPU使用率,是单位时间内CPU繁忙情况的统计,跟平均负载并不一定完全对应。
        • CPU密集型进程,使用大量CPU会导致平均负载升高,此时这两者是一致的;
        • I/0密集型进程,等待1/0也会导致平均负载升高,但CPU使用率不一定很高;
        • 大量等待CPU的进程调度也会导致平均负载升高,此时的CPU使用率也会比较高。
  • kill用于向进程发送信号（常用于结束进程）

网络

netstat ss lsof nc telnet

  • netstat
  • ss
  • lsof
  • nc
  • telnet命令探测远程端口

ssh nohub

  • ssh 通过 SSH 协议远程登录到另一台主机，或在远程主机上执行命令（通信加密）
  • nohub让命令在退出终端或断开 SSH 后仍继续运行（忽略挂断信号 SIGHUP）

安装、构建、执行、部署、发布

java命令

javac java jar javadoc

  • javac用于将Java源代码编译成Java字节码文件（.class文件），以便在Java虚拟机（JVM）上运行
  • java命令用于启动并执行Java程序。使用java命令可以在命令行中直接运行编译后的Java程序
  • javap命令用于查看编译后的Java类文件的信息。它可以解析字节码文件，并显示有关类及其成员的信息
  • jar用于创建和管理JAR格式的归档文件，也可以用于解压缩包和对JAR文件进行签名等操作
  • javadoc用于生成Java文档。它可以分析源代码中的注释，并以HTML格式输出API文档

jps jstack jmap jstat jinfo

  • jps用于列出当前系统中正在运行的Java进程。它可以显示正在运行的Java虚拟机（JVM）以及它们的进程ID
  • jstat用于监视Java应用程序的性能。它可以使用不同的选项打印出Java虚拟机（JVM）中各个内部组件的性能统计数据
  • jstack用于查看Java应用程序的线程信息和调用栈信息。它通常用于分析Java进程出现卡死、死锁等问题时定位问题原因

需要注意的是，jstack命令只能获取Java进程当前的线程信息和调用栈信息，并不能记录历史数据，如果需要跟踪Java进程的历史性能指标，建议使用其他性能分析工具

  • jconsole是Java开发环境（JDK）自带的图形化监视工具，它提供了一个友好的用户界面，用于可视化地监控和管理Java应用程序。它可以实时显示Java应用程序的性能指标、内存使用情况、线程状态等信息，还可以进行远程监控

java 项目管理

mvn jenv

  • mvn 用于构建、测试、打包与管理 Maven 项目依赖
  • jenv 用于管理多个 Java 版本（在不同项目/终端切换 JAVA_HOME 与 java/javac 指向）

管理 Node.js

npm yarn pnpm

  • npm 用于管理 Node.js 项目的依赖与脚本（安装、更新、运行构建等）
  • yarn 用于管理 Node.js 项目依赖与脚本（类似 npm，常用于更快/更稳定的依赖安装）
  • pnpm 用于管理 Node.js 项目依赖与脚本（节省磁盘、安装更快，常用于 monorepo）

包管理

brew

  • brew 用于在 macOS（及 Linuxbrew）安装、更新与管理软件包

nvm

  • nvm 用于管理多个 Node.js 版本（安装、切换、设置默认版本）

Docker 命令

  • docker 用于构建镜像与管理容器（运行、停止、查看日志、进入容器等）

Git命令

  • git 用于版本控制（提交代码、分支协作、拉取合并、查看历史等）

Java问题排查

CPU使用率达100%

  • CPU使用率达100%,top显示的占用率是所有CPU核心占用CPU的数值累加
    • 通过top找到占用率高的进程
    • 通过top -Hp pid找到对应的线程ID
    • 把线程ID转化为16进制printf "0x%x\n" <线程id>，假设它是0x3be
    • 通过命令jstack <进程id> | grep '0x3be' -C5 --color 或者 jstack 163 | vim +/0x3be -找到问题代码
